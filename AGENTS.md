# AGENTS.md

This document defines the boundaries and expectations for AI agents operating in this repository.

Human maintainers own the project direction and review process.
AI assistance is scoped as defined below.

# Core Rule

AI agents may modify production code when explicitly requested by maintainers.

Changes must remain scoped to the requested task and preserve human intent.
Behavioral decisions should be explained clearly when they affect public APIs,
runtime behavior, or long-term maintenance.

All production changes remain subject to maintainer review.

# Allowed Scope for AI Assistance

AI agents may assist with:

* Production code changes requested by maintainers
* Code comments
* GoDoc documentation
* README updates
* Markdown documentation
* Test scaffolding
* Table-driven test case generation
* Expanding test coverage (without altering logic)
* Improving clarity of documentation

AI may suggest improvements outside the requested scope, but should not bundle
unrequested behavioral changes into an implementation.

# Style Enforcement

When assisting with comments, documentation, or tests, AI agents must follow `STYLEGUIDE.md`.

This includes:

* File-per-type structure
* Constructor placement rules
* Named returns (no naked returns)
* Early return patterns
* Grouped var declarations
* No shadowing

AI must not introduce stylistic drift.

# Tests and AI

AI may generate tests that validate existing behavior.

Tests generated by AI must:

* Be readable
* Follow table-driven patterns when appropriate
* Remain explicit and non-clever
* Avoid unnecessary abstractions

# Documentation Expectations

AI may:

* Improve clarity of comments
* Expand GoDoc descriptions
* Add architectural explanations
* Clarify intent and rationale

AI must not rewrite logic under the guise of documentation updates.

# Pull Request Descriptions

All pull request descriptions must follow the structure defined in:

`.github/PULL_REQUEST_TEMPLATE.md`

AI agents must:

* Use the exact section structure from the template
* Preserve checklist items
* Not remove required fields
* Not reorder sections unless explicitly instructed
* Not introduce additional sections unless requested

When generating a pull request description:

* The entire PR description must be wrapped in a fenced Markdown code block.
* No explanatory text may appear outside the code block.
* The output must be ready to copy and paste directly into GitHub.
* The template structure must be preserved.

AI must not simplify or compress the template for convenience.

If unsure, produce less text — never alter structure.

# Change Discipline

Human maintainers retain control over behavioral direction.

AI agents should keep implementation changes narrow, avoid unrelated refactors,
and call out any public API or behavior choices made while completing a task.

# Guiding Philosophy

This repository prioritizes intentional design and explicit authorship.

When in doubt, do less.
Preserve human intent.
Defer behavioral decisions to maintainers.
